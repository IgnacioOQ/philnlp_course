# рЯОУ Philosophy of NLP вАФ LMU Munich (MCMP)

A hands-on, from-scratch course on natural language processing for philosophy students at the **Munich Center for Mathematical Philosophy (MCMP)**, LMU Munich.

We build each model the field has used to represent meaning вАФ from counting word sequences to modern transformers вАФ and at each step ask the philosophical question hiding behind the math: *what kind of meaning does this representation capture, and what does it miss?* No prior NLP background is assumed; Python basics are introduced in the first unit.

---

## рЯІ≠ What the course covers

The course is a guided walk through four ways of representing language, each more expressive than the last:

1. **рЯФ§ N-grams** вАФ meaning as raw co-occurrence statistics. Count things; predict the next token.
2. **рЯО≤ Hidden Markov models** вАФ introduce *latent state*: what we see is generated by what we can't directly observe.
3. **рЯІђ Contextual embeddings** вАФ move from discrete symbols to continuous vectors, and from one-vector-per-word to vectors that depend on context.
4. **рЯ§Ц Transformers** вАФ build self-attention from dot products up to a full encoderвАУdecoder in PyTorch, the architecture behind every modern LLM.

Each unit reflects a different philosophical commitment about what meaning *is* (statistical regularity, hidden structure, geometric similarity, learned context). The progression is the argument.

## вЪЩпЄП Setup

### рЯЦ•пЄП Local (recommended)

The repo uses a local virtual environment named `.venv` running **Python 3.13.7**. All dependencies are pinned in [`requirements.txt`](requirements.txt) at the repo root.

```bash
python3.13 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

> рЯУ¶ **What's in `requirements.txt`?** Core scientific Python (`numpy`, `matplotlib`), `nltk` for Unit 1's n-gram tutorial, `torch` for Unit 4's transformers, and Jupyter (`notebook`, `jupyterlab`, `ipykernel`) for the `.ipynb` units. See the file for line-by-line annotations.

Register the kernel so notebooks pick the right interpreter:

```bash
python -m ipykernel install --user --name transformer-tutorial --display-name ".venv (3.13.7)"
```

> вЪ†пЄП **Heads-up:** do **not** use the system Anaconda Python at `/Users/.../anaconda3` for this course вАФ its bundled PyTorch is broken and will crash the Jupyter kernel. Always run notebooks against the `.venv` kernel.

### вШБпЄП Google Colab

If you don't want to install anything locally, open the notebook of interest in [Google Colab](https://colab.research.google.com/) and run [`setup/colab_setup.ipynb`](setup/colab_setup.ipynb) first вАФ it mounts Google Drive and sets the base paths.

## рЯЪ¶ How to use this course

1. рЯРН Start in [`00_python_intro/`](00_python_intro/) if you've never written Python before.
2. рЯФ§ Move through the units in order: `01_ngrams` вЖТ `02_hmm` вЖТ `03_contextual_embeddings` вЖТ `04_transformers`. Each one motivates the next.
3. рЯУЬ Some units are runnable `.py` scripts (e.g. [`01_ngrams/01_ngrams_tutorial.py`](01_ngrams/01_ngrams_tutorial.py)) вАФ just `python <file>` and read the source alongside the printed output. Others are Jupyter notebooks (the transformer unit) вАФ open them in **Jupyter** or VS Code's notebook view and run cells top-to-bottom.
4. рЯФђ Don't read passively. Change the inputs, break things, see what happens.
5. рЯСА In the transformer unit especially, **actually look at the attention-weight heatmaps** when the notebook prints them. The whole point of attention is that it's visualizable.
